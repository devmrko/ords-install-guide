# 04. HA — 2노드 + OCI Load Balancer + 사설 TLS

## 4.1 아키텍처

```
       Internet (443/80)
              │
              ▼
     ┌──────────────────┐
     │  OCI Load Balancer  │  ← 사설 TLS 인증서로 종단
     │   (Flexible shape) │     HTTP(80) → HTTPS(443) 리다이렉트
     └──────┬─────┬─────┘
            │     │  HTTP 8080 (VCN 내부)
       ┌────▼─┐ ┌─▼────┐
       │ ORDS │ │ ORDS │   ← 둘 다 standalone + systemd
       │  N1  │ │  N2  │
       └───┬──┘ └──┬───┘
           │       │
           ▼       ▼
       ┌──────────────┐
       │     ADB      │  (myadb_high)
       └──────────────┘
```

- ORDS 는 stateless → sticky session 불필요
- LB healthcheck: `GET /ords/_/landing` (HTTP 200)
- TLS 는 LB 에서만 종단, backend 는 평문 HTTP 8080 (VCN 내부)

## 4.2 사전 (네트워크)

- VCN / Public Subnet (LB 용) / Private Subnet (ORDS 노드 용) 분리 권장
- Security List:
  - **LB → ORDS**: ingress 8080 from LB subnet CIDR
  - **인터넷 → LB**: ingress 443 / 80 from 0.0.0.0/0
  - **ORDS → ADB**: egress 1522 to ADB private endpoint (또는 NAT)

## 4.4 LB 구성 — 트랙 A: 웹 콘솔 (수동)

### 4.4.1 사설 인증서 등록 (OCI Certificate Service)

1. **콘솔 메뉴**: Identity & Security → **Certificates**
2. **Create Certificate** 클릭
3. 입력:
   - **Compartment**: ORDS 가 있는 곳
   - **Name**: `ords-private-cert`
   - **Certificate type**: **Imported**
4. **Certificate** 단계에서 PEM 들 업로드:
   - **Certificate (PEM)**: `cert.pem`
   - **Private key (PEM)**: `privkey.pem` (암호화 안 된 것)
   - **Certificate chain (PEM)**: `chain.pem` (중간 CA, 있으면)
5. **Rules**: 만료 알림은 기본값 유지
6. **Create** → 생성된 인증서의 **OCID 복사**

### 4.4.2 Load Balancer 생성

1. **Networking → Load Balancers → Create Load Balancer**
2. Type: **Public** (또는 Private), Shape: **Flexible** (10–100 Mbps)
3. **Backends**: Add Backends → ORDS 노드 IP × 2, port `8080`
4. **Health Check**:
   - Protocol: HTTP
   - Port: `8080`
   - URL path: `/ords/_/landing`
5. **Listener**:
   - Protocol: **HTTPS**, Port `443`
   - **Certificate**: 4.4.1 에서 만든 OCID 선택
6. **Create**
7. (선택) HTTP 80 → HTTPS 301 리다이렉트 ruleset 추가

5분 후:
```bash
curl -k https://<LB_PUBLIC_IP>/ords/_/landing
```

## 4.5 LB 구성 — 트랙 B: Terraform (권장)

`terraform/main.tf` 가 위 단계를 전부 코드화. 인증서도 같이 import 함.

### 4.5.1 준비
```bash
# 컨트롤 머신에서 (ORDS 노드 아니어도 됨, OCI CLI만 있으면 됨)
oci setup config            # ~/.oci/config 생성
terraform version           # 1.3+
```

### 4.5.2 사설 인증서 PEM 위치
`.env` 에:
```bash
TLS_CERT_PEM=/etc/ords/tls/cert.pem
TLS_KEY_PEM=/etc/ords/tls/privkey.pem
TLS_CHAIN_PEM=/etc/ords/tls/chain.pem   # 없으면 빈 값
OCI_CERT_NAME=ords-private-cert
OCI_CERT_OCID=                          # 비워두면 새로 import
```

### 4.5.3 실행
```bash
./run.sh ha-tf plan
./run.sh ha-tf apply
./run.sh ha-tf output
# lb_public_ip = [ "129.x.x.x" ]
# cert_ocid    = "ocid1.certificate.oc1..xxxx"
```

### 4.5.4 동작 검증
```bash
LB_IP=$(./run.sh ha-tf output -json | jq -r '.lb_public_ip.value[0]')
curl -k https://$LB_IP/ords/_/landing
```

### 4.5.5 인증서 회전(rotation)
- 새 PEM 으로 갈아끼우려면: `.env` 의 PEM 경로 갱신 후 `./run.sh ha-tf apply`
- Terraform 이 `oci_certificates_management_certificate` 의 새 버전 생성 → LB listener 가 자동으로 최신 사용

### 4.5.6 정리
```bash
./run.sh ha-tf destroy
```
주의: import 한 인증서도 같이 지워짐. 다른 서비스가 참조 중이면 `terraform state rm` 으로 분리 관리.

## 4.6 운영 팁

- **롤링 재시작**: LB 에서 한 노드 drain (backend offline) → `systemctl restart ords` → 응답 정상 확인 → 반대편 동일
- **세션 한도**: `JDBC_MAX_LIMIT × 노드수 ≤ ADB sessions`
- **로그 집계**: 노드별 `journalctl -u ords` 와 `/etc/ords/config/logs/` → 중앙 로그(예: OCI Logging) 로 보내는 것 권장
- **APEX**: 이 가이드 범위 밖. 같은 ORDS 인스턴스에 추가 가능하지만 별도 챕터로 분리 추천
